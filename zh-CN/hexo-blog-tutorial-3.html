HEXO 静态博客教程（三）：HEXO 博客 DIY 和日常运维（进阶篇）

发表于 2017-09-23 | 更新于 2017-09-25 | 分类于 IT | |

在本地博客文件+服务器托管完成后，您就完成了建站的基础工作，博客就算正式上线运行了。但是一篇文章，从 idea 到发布到博客，需要经历怎样的过程呢？我该如何去定制博客的方方面面？这一章节将主要围绕 Hexo 常用指令、定制博客信息 、Hexo 站点配置和主题配置等来展开。进阶教程的篇幅将比基础篇长一些，请您善用小标题索引来快速找到所需的信息。

一篇博文的旅程

  1. Markdown 语言写出一个内容文件 article.md。

  2. （可选）清除缓存文件 （ db.json）和已生成的静态文件 （public）。通常，在 hexo g 前之前使用该清除命令是个好习惯，但一般可以不用。在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

  3. Hexo 把 source 文件夹里的博客内容、样式、图片、媒体等整合为一个 网页文件 html，并且按照预设的分类（Categories)、标签（Tags）等自动归档到不同的文件夹里。所有将发布的博客内容，即 public 这个文件夹内的内容。

  4. （可选）在本地 http://localhost:4001 预览发布结果。

  5. 提交代码到服务器

常用 Hexo 指令

指令 作用 重要程度
hexo n [layout] <title> 新建一篇文章 ⭐
hexo g 生成静态文件 ⭐⭐⭐
hexo d 部署网站 ⭐⭐⭐
hexo s -p 4001 启动本地服务器（4001端口） ⭐⭐
hexo clean 清除缓存和静态文件 ⭐⭐
hexo d -g
hexo g -d		 生成文件后立即部署 ⭐

站点配置

在博客安装的根目录下的站点配置文件 \_config.yml 是使用 YAML 语言写的。我们不会 YAML，但没关系，系统自带的记事本就可以进行编辑。若您想使用一款功能比较强大的轻量级编辑器，可以参考下方的文章；若有选择恐惧症，建议选择和我一样的 Notepad++ 即可。

  • 10个常用的代码文本编辑器/ IDE（知乎专栏 - 路人甲）

    https://zhuanlan.zhihu.com/p/21434126/

  • 推荐15款很棒的文本编辑器（开源中国社区 - 红薯）

    http://www.oschina.net/news/17858/15-best-text-editors/

下面即是我自己的站点配置文件，相关解释直接写在文本注释中（目前我正在研究折叠大段代码块的方法，所以暂时网页会显得很长，请谅解）。由于该文本很长，我进行了删减，只在这里展示一些比较常用的设置。需要注意以下几点：

  • 以 “#” 开头的即为注释，被注释掉的文本将不被执行。你可以通过注释掉某一行，来关闭某个功能。
  • 每个参数后面是一个冒号，紧跟着一个空格，然后才是参数值，比如 title: SKRTEC 博客。
  • 符号应一律使用英文半角。

主题配置

挑选一款主题

Hexo 默认的主题并不是很好看，接下来我们去物色一款喜欢的主题。这边给您带来了一些参考网页。

  • Hexo 官网主题浏览：

    https://hexo.io/themes/

  • 知乎 - 有哪些好看的 Hexo 主题？– 家顺张的回答：

    https://www.zhihu.com/question/24422335/answer/46357100

下载一款主题

选好了主题，就把他下载下来并且存放到博客的主题文件夹下面，并且在站点配置里面修改theme字段的值即可。

例如，本站目前使用的是 Next 主题，制作者是 iissnan，这是目前最受欢迎的主题。主题配色以黑白灰为主，分区明快，线条简洁。功能上，内置多语言支持、文本高亮、标签书写。这是目前集成第三方库最完善的主题之一（即在主题配置文件里就可以快速引用插件），包括评论系统、访问量统计分析、内容分享、内置搜索、公式排版等。如果你目前还不知道选择什么主题，可以暂时先使用 Next 来学习如何配置主题，今后更换也十分简单。

  • hexo-next 官网：

    http://theme-next.iissnan.com

  • hexo-next 发布页面（稳定版本下载）：

    https://github.com/iissnan/hexo-theme-next/releases

在发布页面找到 Download Source Code，将源码包下载并解压到自己的博客文件夹 \themes\下即可，如下图所示。next 文件夹打开后应该直接就是各种文件。

去上面所讲的站点配置文件里，保持 theme字段的值和上图你建立的文件夹名字一致即可。

主题配置文件 \themes\~\_config.yml

在 themes 目录下你的主题文件夹里找到 _config.yml，注意这和根目录下的站点配置文件 \_config.yml不是同一个文件。由于该文本很长，我进行了删减，只在这里展示一些比较常用的设置。
