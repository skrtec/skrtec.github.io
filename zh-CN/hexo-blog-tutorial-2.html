HEXO 静态博客教程（二）：将博客托管到服务器（基础篇）

发表于 2017-09-23 | 更新于 2017-09-25 | 分类于 IT

上一章节中我们已经在本地建立了完整的博客系统，下一步要做的就是找一家提供网页托管的厂商来存放我们的博客。目前 GitHub、 Coding.net（原 GitCafe）、 七牛云、GitLab 等都提供免费的静态页面托管服务。这里我们以 GitHub Pages 为例，毕竟它是程序猿界的 Facebook 。其他网站的处理方式是几乎完全一样的，您可以自己尝试。

在最后的扩展教程里，我们还会展示如何将自己的博客同时部署到多个托管网站（镜像站点），但访问他们都只需要同样的域名；并根据用户线路，使用 DNS 将其指向不同的镜像，加快不同区域访问博客的速度。

创建一个 GitHub 账号

在 GitHub 网站注册一个账号 https://github.com/join

这应该非常容易，记得取一个骚气的名字！值得一提的是， GitHub 有且仅有英文界面。Coding.net 是模仿 GitHub的中文站点，在中国的访问速度远快于后者，有这方面需求的您可以使用 Coding.net。

创建一个代码仓库

我们需要创建一个项目仓库来专门存放博客文件。点击 New Repository 来创建一个新的仓库，名称必须是 <你的GitHub名字>.github.io，比如 skrtec.github.io。

选择 Public 选项，仓库内容对所有人可见，否则别人看不到你的网页。

README ，\.gitignore 和 license 不需要添加，完成创建即可。

配置 SSH 密钥

为什么需要配置 SSH 密钥呢？因为我们今后文件的上传都是批量的，总不能每次都打开网页版来上传吧。前一章的 Git Bash 只需要一个命令就可以完成对我们在 GitHub 里文件的更新。因此，需要一把密钥来对双方进行认证，保证 git 操作的安全性。

先找到我们的 ssh 文件夹，win系统通常在 <user>\.ssh。当然更简单的方法就是在 Git Bash 里使用如下两条命令。

如果显示你已经有密钥文件在里面，那么可以跳过下一步的生成密钥，直接去后面的提交公钥部分。如果显示没有密钥，使用你自己的邮箱来生成一个密钥。

  • 推荐的做法是一路回车到完成。中间会出现让你输入密码，这个密码的作用是每次提交文件到 GitHub 的时候让你输入这个密码来认证（不是 GitHub 的密码）。推荐不输，这样可以直接提交代码。

在 GitHub网站的 Settings - SSH and GPG keys （ https://github.com/settings/keys ）里创建一个 New SSH Key，Title 任意。打开电脑上 <user>\.ssh 文件夹找到后缀为 .pub 的公钥，用记事本打开复制里面的内容并且贴到Key里面去。

  • 记住上传是 .pub 的公钥，私钥永远是存在自己机器上的。

  • 我为什么不需要使用 GitHub 的账号密码就能上传博客？

    因为上面我们已经进行了 SSH 密钥的配置，这远比账号密码要复杂和安全。每次进行 git 操作时都会自动使用密钥进行认证的。
