HEXO HEXO 静态博客教程（三）：HEXO 博客 DIY 和日常运维(进阶篇)

发表于 2017-09-23 | 分类于 IT | |

在本地博客文件+服务器托管完成后，博客就算正式上线运行了。我们还需要掌握一定的博客管理维护技巧，才能进行正常的发帖、修改、删帖等动作。这一章节将主要围绕 Hexo 常用指令、 Hexo 站点配置和主题配置来展开。

一篇博文的旅程

  1. Markdown 语言写出一个内容文件 article.md。

  2. （可选）清除缓存文件 （ db.json）和已生成的静态文件 （public）。在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

  3. Hexo 把 source 文件夹里的博客内容、样式、图片、媒体等整合为一个 网页文件html，并且按照预设的分类（Categories)、标签(Tags)等自动归档到不同的文件夹里。所有将发布的博客内容，即 public 这个文件夹内的内容。

  4. （可选）在本地 http://localhost:4001 预览发布结果。

  5. 提交代码到服务器

常用 Hexo 指令

指令 作用 重要程度
hexo n [layout] <title> 新建一篇文章 ⭐
hexo g 生成静态文件 ⭐⭐⭐
hexo d 部署网站 ⭐⭐⭐
hexo s -p 4001 启动本地服务器（4001端口） ⭐⭐
hexo clean 清除缓存和静态文件 ⭐⭐
hexo d -g
hexo g -d		 生成文件后立即部署 ⭐
