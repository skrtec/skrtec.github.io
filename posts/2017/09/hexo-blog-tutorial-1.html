HEXO 静态博客教程（一）：HEXO 本地环境搭建（基础篇）

发表于 2017-09-22 | 分类于 IT | |

在序章中我们提到，让博客上线运行，最少只两大步：本地环境搭建+上传到托管站点。这一节中我们具体介绍如何在自己的电脑上用 Hexo 快速搭建一个博客系统。您将需要的工具有 Node.js 和 Git。

安装 Node.js 和 Git

Hexo 是一个快速、简洁且高效的博客框架，它需要 Node.js 和 Git 作为运行环境，请根据自己电脑的 32位/64位 架构来选择对应的版本。

Node.js 官方下载页面： https://nodejs.org/zh-cn/

  • 建议勾选 Add to PATH

Git 官方下载页面： https://git-scm.com/downloads

  • 勾选 Git Bash Here
  • 选择 Use Git from the Windows Command Prompt
  • 其他设置可以全部使用默认

上述建议将允许 Node.js 和 Git 指令添加到系统变量环境，这将允许您在命令行中直接调用它们的指令。若未勾选，您也可以在通过指定路径来运行，或随时将其添加到 PATH。若您勾选了 Git Bash Here, 在下文任何需要命令行的地方，都可以通过在任意文件夹右键使用 Git Bash Here，即可在当前路径打开 Git 命令行窗口，也是十分的方便。

使用 Hexo 的 Server 指令将博客站点部署到本地服务器上。目前我们以自己的电脑作为站点发布的服务器，还不能被其他人访问到。在下一章我们搞定了托管服务后，就可以将网站发布到外部服务器上，被互联网所访问到。当然，你仍然可以随时使用这个指令来预览博客，作为每一次发布前的测试。

小贴士：若不写 “4001” ，Hexo 将默认以 4000 作为端口，4000端口被占用的可能性比较大。若出现端口被占用无法正常浏览的情况，请将端口号换成任意另外一个其他数字即可。

打开浏览器，访问 http://localhost:4001/ 或 http://127.0.0.1:4001/，看到你的 Hexo 博客了吗？
